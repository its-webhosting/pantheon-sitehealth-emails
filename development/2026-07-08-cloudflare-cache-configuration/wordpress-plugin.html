Installing the umich-cloudflare plugin for WordPress

A Cloudflare plugin for WordPress is necessary to allow WordPress to:

  • Tell Cloudflare what pages and assets to cache and what ones not to cache. This is done using HTTP caching headers.
  • Tell Cloudflare how long to cache different pages and assets. This is done using HTTP caching headers.
  • Tell Cloudflare when WordPress content has been modified so Cloudflare can serve the new version of the content to visitors instead of continuing to serve the old version.
  • Allow the WordPress administrator to tell Cloudflare to purge all content in its cache for the website so that visitors will get new (updated) content rather than old (cached) content.

ITS supports the umich-cloudflare plugin for integrating WordPress with Cloudflare. This page documents how to install and configure the umich-cloudflare plugin.

  • ITS does not support other WordPress plugins that can integrate with Cloudflare, but for information about them consult Other Cloudflare plugins for WordPress below.
  • ITS recommends against using a WordPress website that is behind Cloudflare without a Cloudflare plugin, but if you want to do this, refer to the section Using WordPress without a Cloudflare integration plugin below.
Note: Please ensure you are using the latest version of the umich-cloudflare plugin. You should use version 1.1.0 or newer.

Get a Cloudflare API token

Refer to the documentation Create a Cloudflare API Token to use with a Cloudflare Content Management System module/plugin for instructions on obtaining a Cloudflare API token that can be used with the umich-cloudflare WordPress plugin.

Cloudflare API tokens are usable for all websites in the Cloudflare zone for which the token was created. If you already have a Cloudflare API token for the zone your website is in, you can use that token for all websites in the zone.

Install the umich-cloudflare plugin

The umich-cloudflare plugin is not available through the worldwide WordPress Plugins Directory. This means that you cannot install umich-cloudflare by searching for it under Plugins in your WordPress administrative dashboard. Instead, you will need to use one of the two special options below to install the umich-cloudflare plugin.  

Installation option 1: WP CLI

If you have WP CLI installed for your WordPress website, install and activate the latest release of the umich-cloudflare plugin using the following commands:

DOWNLOAD_URL=$(curl -s \
    "https://api.github.com/repos/its-cloudflare/umich-cloudflare/releases/latest" | \
    jq -r '.assets[] | .browser_download_url')

wp plugin install --activate $DOWNLOAD_URL

 

Installation option 2: Download then upload

Note: If your site is hosted on AFS Virtual Web Hosting, you will need to have the ssh-sftp-updater-support plugin installed, activated, and configured in order to use this installation option. Please see ITS Knowledge Article #3148, Updating ITS Web Hosting WordPress Sites Within the WP Dashboard, for more information.
  1. Download the plugin’s zip file to your local computer from https://github.com/its-cloudflare/umich-cloudflare/releases/latest

    Important: Make sure you do not download the source code. The source code has to be built before it can be used on a site.

     
  2. Log in to your WordPress dashboard
  3. Go to Plugins -> Add New ➊. Do not search for the plugin. Instead, click the “Upload Plugin” button ➋:
  4. Click the “Choose file” button
  5. In the file box, select the zip file that you downloaded for the plugin.
  6. In your WordPress dashboard, click the “Install Now” button:
  7. Click the “Activate Plugin” button:

Configure the umich-cloudflare plugin

After the umich-cloudflare plugin is installed and activated, it will appear in the WordPress dashboard left-side menu and on the Installed Plugins page as “University of Michigan: Cloudflare Cache”. In the dashboard menu, under “Settings” it will appear as “U-M: Cloudflare”.

  1. U-M Cloudflare Cache Settings
    1. From the WordPress administration dashboard, go to Settings -> U-M Cloudflare.

Configure the plugin using the following settings, adjusting TTL settings according to your own site needs:

Setting NameSetting Value
API Key{Cloudflare API Token}
Zone(auto-filled after API Token entered above)
Default Page TTL31536000 (should be the default value)
Default Browser TTL300 (Default: 0)
Default Static TTL31536000 (should be the default value)

Press “Save Changes” when done.

WordPress should now be fully integrated with Cloudflare.

Note: If your site is hosted on Pantheon, you will also need to have the Pantheon Advanced Page Cache plugin installed, activated, and configured in order to use this installation option. Please see the Pantheon documentation, Pantheon WordPress Cache Plugin Configuration, for more information.

Other Cloudflare plugins for WordPress

WordPress plugin information as of August 2024. ITS cannot provide support or assistance for any of these plugins.

  • Official Cloudflare plugin
    • Requires write access to Cloudflare domain settings. Any WordPress administrator can override the Cloudflare domain settings for all other websites in the Cloudflare domain.
    • Could be used with a U-M Basic Cloudflare domain, especially if the domain has only one website. Should not be used for U-M Self Service Cloudflare. Does not work with U-M Turnkey Cloudflare.
  • Super Page Cache for Cloudflare
    • Requires write access to Cloudflare domain settings. Any WordPress administrator can override the Cloudflare domain settings for all other websites in the Cloudflare domain.
    • Could be used with a U-M Basic Cloudflare domain, especially if the domain has only one website. Should not be used for U-M Self Service Cloudflare. Does not work with U-M Turnkey Cloudflare.
  • WP Rocket
    • Does not support Cloudflare tokens. Requires a Global API Key, instead, which belongs to a particular user and gives all WordPress administrators the same permissions in Cloudflare that the user has. The website may break when that user changes jobs or leaves the university.
    • Requires write access to Cloudflare domain settings. Any WordPress administrator can override the Cloudflare domain settings for all other websites in the Cloudflare domain.
    • Using WP Rocket for regular WordPress caching is OK, but do not turn on Cloudflare integration.
  • UMich Hummingbird plugin for WordPress
    • Custom U-M build of the Hummingbird plugin by ITS, based on the WPMU DEV Hummingbird plugin.
    • No longer supported by U-M ITS, this plugin has been replaced by the umich-cloudflare plugin.
    • All sites still using the UMich Hummingbird plugin must replace it with the umich-cloudflare plugin as soon as possible, since the UMich version will no longer be updated with future releases.

Using WordPress without a Cloudflare integration plugin

WordPress can work with Cloudflare without any Cloudflare plugin for WordPress, but because the university has changed Cloudflare’s configuration to cache all pages, changes you make to website content will not be visible right away.

ITS recommends against not using a Cloudflare integration plugin, but if you need to run a WordPress website behind Cloudflare without one, there are two options:

  • Configure the HTTP caching headers to only cache content for a short time, for example, 15 minutes. You can do this using a WordPress caching plugin that does not support Cloudflare integration, or you can configure your web server to do it.
  • Alternatively, Self Service and Basic customers can turn off the “UM default: Cache pages for non-logged-in users” cache rule for their Cloudflare domain. If you do this, Cloudflare will no longer protect any website in your Cloudflare domain against surges in traffic, and websites in that domain could become slow, show error messages to visitors, or go down entirely during high-traffic periods.
